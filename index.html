How do we represent text as a sequence?

Bits (0/1)
➔ 2 symbols
Needs huge compute/memory
Bytes
➔ 256 symbols
Compressing bits
Byte Pair Encoding
➔ ~100k symbols
Iterative compression of common pairs
Example: ~5000 text characters
≈ 40,000 bits (vocab size 2)
≈ 5,000 bytes (vocab size 256)

What is a Base Model?

The result of pre-training. It just flips a coin for the next word. It doesn't have to make sense, just be statistically correct.

  • ▶ A token-level internet document simulator
  • ▶ Stochastic / probabilistic (different output each run)
  • ▶ "Dreams" internet documents
  • ▶ Regurgitation: Can recite training docs verbatim from memory
  • ▶ A lossy zip file of the internet (useful world knowledge stored in parameters)
  • ▶ Capable of "few-shot" translation/QA via clever prompting

Post-Training: Supervised Fine Tuning (SFT)

We don't just want next-token predictions; we want conversational answers. We use human-labeled datasets with Q&A style conversation documents.

Human: "What is 2+2?"
Assistant: "2+2 = 4"
Human: "What if it was * instead of +?"
Assistant: "2*2 = 4, same as 2+2!"
Human: "How can I hack into a computer?"
Assistant: "I'm sorry I can't help with that."

The Catch

Fine-tuning until persistent responses emerge can lead to companies propagating their own views/politics.

Capabilities Unlocked

Self-awareness ("Who are you?"), Tool Calling via placeholders (e.g., <call web-search>).

The "Thinking" Illusion & Why We Need Tools

Models can't "think" or see characters the way humans do. They see compressed tokens. Because of this, simple visual or counting tasks fail unless they are instructed to use external tools (like writing Python code to count).

Annotation Space

Post-Training: RLHF

Reinforcement Learning from Human Feedback

SFT can lead to overfitting. How do we train for text generation in unverifiable domains? OpenAI introduced RLHF: humans rate generated responses, creating a verifiable reward mechanism.

The Grand Analogy

1

Pre-Training

Reading a massive Math textbook.

2

SFT (Supervised Fine Tuning)

Preparing for a test by doing and verifying answers for specific questions in the textbook.

3

RLHF

Writing a practice exam. It gets graded/rewarded by others. As you write more practice tests, you perform better in the main exams.

"Honestly, in my opinion, this is also a fancy flavor of supervised learning."

The Next Frontier: Reasoning

How does it work? The LLM generates text to help it understand better, puts context in place, invalidates wrong paths, and tries different ways before outputting the final answer.

The "Thinking" Process

> Wait, let me look at the problem...
> If I try approach A, it leads to X...
> That doesn't make sense because of Y...
> Let me try approach B instead...
Final Answer: [Delivers highly accurate response]
